## Component Explanations (Interview-Ready)

| Component | What It Does | How to Explain |
|-----------|-------------|----------------|
| **API Gateway** | Front door for all requests | "Validates JWT tokens to identify tenants, applies per-tenant rate limits, routes to correct model." |
| **Load Balancer** | Traffic distribution | "Distributes requests across healthy replicas. Checks health every 10 seconds." |
| **Model Servers** | Run ML predictions | "Each model in its own K8s Deployment. HPA auto-scales based on load." |
| **Feature Store** | Pre-computed user data | "Features prefixed with tenant_id for isolation. Response time <5ms." |
| **Prediction Cache** | Avoid duplicate work | "Cache identical predictions. Keys include tenant_id. Saves ~40% compute." |
| **Model Registry** | Version control | "Enables rollback to previous version in under 2 minutes." |
| **Monitoring** | Watch everything | "Tracks latency (p50/p95/p99), errors, drift. Auto-rollback if error rate >5%." |
